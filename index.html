What is it?

Ixgenup is a utility that posts data to web servers for testing.

What's the story behind this?

I developed it in the interest of having a simple way to test web app security. Being a web app developer myself, I know security and XSS is a big deal when making your web app. That's why the testing phase is crucial. Ixgenup isn't meant to replace manual testing altogether, but at least help in some way.

How does it work?

To make a new request, type "new".
Then you will enter a URL.
Next, you will enter whether your site has form parameters or not. If it does, you can add a new param with "nparam". You can add parameters until you are done, in which case you type "pdone". If your site does not have form parameters or you wish to type it out manually, you will say no and it will allow you to put the body.
NOTE: The parameters and body as of now only accept strings, and all inputs will be interpreped as strings.
Once this is done, it will make a POST request with the link and will return the web server response.

What was it made in?

It was made in the Rust programming languages using packages by other open-source developers. Ixgenup specifically uses reqwest for posting data to sites.
